# 加密数据查询与管理系统 - 测试计划与验收矩阵

- 文档版本：1.0
- 修订日期：2026-03-20
- 依据文档：SRS v1.0（草案）

## 1. 测试目标

1. 验证系统功能满足 SRS 全部 P0/P1 条款。
2. 验证安全机制：加密存储、权限控制、审计可追溯。
3. 验证性能目标：百万级数据下查询和导入达到目标阈值。
4. 输出可审计的 UAT 签署材料。

## 2. 测试范围

## 2.1 In Scope

- 用户认证与 RBAC
- XLSX 导入（含大文件）
- 加密与盲索引
- 基础与高级查询
- 数据删除与清空
- 查询配额
- 审计日志
- 非功能项（性能、可靠性、部署可用性）

## 2.2 Out of Scope

- 对外开放 API 网关
- 多租户隔离验证
- 云端托管安全策略

## 3. 测试策略

| 层级 | 目标 | 负责人 | 通过标准 |
|---|---|---|---|
| 单元测试 | 算法正确性与异常分支 | 后端开发 | 核心模块覆盖率建议 >= 80% |
| 集成测试 | 模块协作与事务一致性 | QA/后端 | 核心链路全部通过 |
| API 合同测试 | 接口字段与错误码一致性 | QA/前后端 | OpenAPI 用例 100% 对齐 |
| 安全测试 | 权限绕过、注入、XSS/CSRF 基线 | 安全/QA | 无高危漏洞 |
| 性能测试 | 查询 P95、导入吞吐、资源占用 | QA/后端 | 满足 NFR 指标 |
| UAT | 业务可用性与流程闭环 | 业务代表 | UAT 用例全部通过并签署 |

## 4. 测试环境与数据

## 4.1 环境基线

- OS：macOS/Linux（与生产同架构优先）
- Python：3.8+
- DB：SQLite 3.x（WAL）
- 浏览器：Chrome 最新稳定版
- 网络：支持离线运行

## 4.2 数据集

| 数据集 | 规模 | 用途 |
|---|---|---|
| DS-SMALL | 1,000 行 | 功能冒烟、回归 |
| DS-MEDIUM | 100,000 行 | 集成与稳定性 |
| DS-LARGE | 1,000,000 行（约 300MB XLSX） | 性能与容量验证 |

## 5. 入口/退出标准

## 5.1 入口标准

1. 需求基线冻结（SRS 与 API 草案已评审）。
2. 代码完成自测并通过静态检查。
3. 测试环境与数据准备完成。

## 5.2 退出标准

1. P0 缺陷全部关闭，P1 缺陷有明确处置并获批准。
2. 核心测试用例通过率 100%。
3. 性能指标达到 NFR，或具备书面豁免与整改计划。
4. UAT 签署完成。

## 7. 缺陷分级与处理 SLA

| 级别 | 定义 | 响应时限 | 关闭目标 |
|---|---|---|---|
| P0 | 安全漏洞/核心功能不可用/数据错误 | 4 小时内 | 24 小时内修复或回滚 |
| P1 | 主要功能受影响但有绕行方案 | 1 个工作日 | 3 个工作日内 |
| P2 | 一般缺陷或 UI 问题 | 2 个工作日 | 版本内关闭 |
